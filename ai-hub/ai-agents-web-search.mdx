Botky ahora incluye soporte **nativo** para **Base de conocimiento con IA**, **servidores MCP** y **búsqueda web** dentro de los Agentes de IA — sin necesidad de soluciones alternativas complicadas.

---

## 1. Configurar la Base de conocimiento de IA

<Steps>
  <Step title="Habilita el proveedor OpenAI">
    Conecta OpenAI en **Integraciones → OpenAI**. Esta conexión es necesaria para crear y gestionar los *vector stores* y archivos.
  </Step>
  <Step title="Prepara los archivos para el Vector Store">
    Los archivos deben tener una extensión válida como `.pdf`, `.xls` o `.doc`. El sistema no puede procesar archivos sin extensión. Puedes subirlos desde tu computador o indicar una URL.
  </Step>
  <Step title="Crea el Vector Store">
    En la sección **Vector Store**, crea uno nuevo con: un **nombre**, una **fecha de expiración** opcional (usa `0` para que sea permanente) y los **archivos** seleccionados de tu colección.
  </Step>
  <Step title="Construye la Base de conocimiento">
    Crea una **Base de conocimiento** indicando: nombre, descripción (opcional) y los **Vector Stores** asociados. Al terminar, queda disponible para tus Agentes de IA.
  </Step>
</Steps>

---

## 2. Usar el modelo OpenAI - Responses dentro de un Agente

<Steps>
  <Step title="Configuración inicial">
    Dentro de un Agente de IA, ve a **Ajustes → Modelos** y selecciona **OpenAI - Responses**. El modelo por defecto es **GPT-4.1**.
  </Step>
  <Step title="Conecta las herramientas">
    Con OpenAI - Responses puedes conectar directamente los **Servidores MCP** y la **Base de conocimiento** al Agente.
  </Step>
</Steps>

### Búsqueda web (Web Search)

La búsqueda web permite buscar directamente en sitios web desde el Agente — útil cuando no tienes servidores MCP ni base de conocimiento disponibles.

**Formato para restringir dominios:**
- Valores separados por comas: `domain.com, sub.domain.com`
- **Omite** `https://` o `http://`
- Ejemplo: `google.com, app.google.com`

**Limitaciones según el modelo:**

| Modelo | Búsqueda web | Restricción de dominio |
|---|---|---|
| GPT-4.1-mini | Sí | No |
| GPT-4.1-nano | Sí | No |
| GPT-4.1 | Sí | Sí |

---

## Nota sobre el uso de GPT-5 (no recomendado)

<Card type="warning">
  1. Su razonamiento profundo produce respuestas más lentas y más errores de timeout.
  2. Requiere un mínimo de **2.000 tokens por respuesta** por el razonamiento extendido (mayor uso de tokens = mayor costo).
  3. Reserva GPT-5 para tareas de razonamiento avanzado; usa modelos más ligeros para consultas simples.
</Card>
