Los **Agentes de IA** incorporan mejoras importantes que facilitan construir agentes más potentes y confiables. Con soporte **nativo** para **Base de conocimiento con IA**, **servidores MCP** y **búsqueda web**, ya no necesitas soluciones alternativas complicadas ni funciones manuales.

## 1. Configurar la Base de conocimiento de IA

### Habilita el proveedor OpenAI

Asegúrate de que el proveedor OpenAI esté conectado en tu espacio de trabajo. Es necesario para crear y conectar los *vector stores* y archivos. Ve a **Integraciones → OpenAI**.

<Frame caption="Conexión del proveedor OpenAI en Integraciones">
  <Placeholder />
</Frame>

<Steps>
  <Step title="Preparar los archivos para el Vector Store">
    Los archivos deben tener una extensión válida: `.pdf`, `.xls`, `.doc`, etc. Sin la extensión correcta, el sistema no puede leerlos ni indexarlos, y los archivos inválidos causan errores o son rechazados. Puedes subirlos desde tu computador o indicar la URL del archivo.

    <Frame caption="Subida de archivos para el Vector Store">
      <Placeholder />
    </Frame>
  </Step>
  <Step title="Crear el Vector Store">
    Ve a la sección **Vector Store**. Crea uno, asígnale un nombre y define una fecha de expiración (opcional, para stores temporales; `0` = sin expiración). Selecciona los archivos que ya subiste en la sección de Archivos.

    <Frame caption="Creación de un Vector Store y selección de archivos">
      <Placeholder />
    </Frame>
  </Step>
  <Step title="Construir la Base de conocimiento">
    Después de crear el vector store, crea una **Base de conocimiento**. Ingresa el nombre, la descripción (opcional) y selecciona los Vector Stores asociados. Una vez creada, puedes usarla en los Agentes de IA.

    <Frame caption="Construcción de la Base de conocimiento">
      <Placeholder />
    </Frame>
  </Step>
</Steps>

## 2. Configurar el modelo OpenAI - Responses dentro de un Agente

<Steps>
  <Step title="Configurarlo">
    Dentro de un Agente de IA, ve a **Ajustes → Modelos**. Selecciona **OpenAI - Responses** y elige el modelo deseado. El valor por defecto es **gpt-4.1**.

    <Frame caption="Selección de OpenAI - Responses en los ajustes del agente">
      <Placeholder />
    </Frame>
  </Step>
  <Step title="Usar herramientas con OpenAI Responses">
    Ahora puedes conectar directamente los **Servidores MCP** y la **Base de conocimiento** al Agente. No hay necesidad de disparar funciones o acciones de IA separadas para ellos. Esto reduce errores y complejidad.

    <Frame caption="Conexión de MCP y Base de conocimiento al agente">
      <Placeholder />
    </Frame>
  </Step>
  <Step title="Integración de búsqueda web">
    Activar la **búsqueda web** te permite buscar directamente en un sitio web desde el Agente. Es útil si no tienes servidores MCP ni base de conocimiento: el agente obtiene respuestas directamente de la web.

    **Restringir dominios:**
    - Formato: `domain.com, sub.domain.com` (separados por comas).
    - Excluye `https://` o `http://`.
    - Ejemplo: `google.com, app.google.com`

    <Frame caption="Configuración de búsqueda web y restricción de dominios">
      <Placeholder />
    </Frame>
  </Step>
</Steps>

### Limitaciones por modelo para la búsqueda web

| Modelo | Búsqueda web | Restricción de dominio |
|---|---|---|
| GPT-4.1-mini | Sí | No |
| GPT-4.1-nano | Sí | No |
| GPT-4.1 (por defecto) | Sí | Sí |

## Nota sobre el uso de GPT-5 (no recomendado)

<Card type="warning">
  1. GPT-5 hace razonamiento profundo, lo que produce respuestas más lentas y más errores de timeout.
  2. Requiere más tokens por el razonamiento extendido. Se recomienda un mínimo de **2.000 tokens por respuesta**. (Mayor uso de tokens = mayor costo.)
  3. Usa GPT-5 solo para tareas de razonamiento avanzado; deja las tareas simples en modelos más ligeros por eficiencia.
</Card>
