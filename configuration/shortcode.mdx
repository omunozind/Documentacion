The `[botky_panel]` shortcode lets you embed the full Botky partner dashboard anywhere on your WordPress site — ideal for giving partners a branded front-end portal without granting them access to the WordPress admin area. Drop it into any page and your partners get a self-service hub for workspaces, clients, billing, and settings, all under your own domain and branding.

## Basic Usage

```text
[botky_panel]
```

Paste this shortcode into the content area of any WordPress page, post, or widget that supports shortcodes. No parameters or additional configuration are required.

## Behavior by Login State

The shortcode renders differently depending on whether the visitor is logged in to WordPress:

<CardGroup cols={2}>
  <Card title="Logged-In Users" icon="circle-check" href="/dashboard/overview">
    The full Botky dashboard renders inline. Partners can manage workspaces, view and create clients, handle billing, and configure their API credentials — all without leaving your site.
  </Card>
  <Card title="Guest Users (Not Logged In)" icon="circle-xmark" href="/configuration/api-credentials">
    A clean login prompt is displayed with a **Log In** button. After the user authenticates, WordPress redirects them back to the dashboard page automatically.
  </Card>
</CardGroup>

## Recommended Setup

<Steps>
  <Step title="Create a new WordPress page">
    In your WordPress admin, go to **Pages → Add New**. Give the page a clear name such as "Partner Dashboard".
  </Step>
  <Step title="Add the shortcode">
    In the page content editor, type or paste `[botky_panel]`. No block wrapper or additional markup is needed.
  </Step>
  <Step title="Publish the page">
    Click **Publish** to make the page live on your site.
  </Step>
  <Step title="Share the page URL with your partners">
    Send the page URL to your reseller partners. They will use this link as their day-to-day portal.
  </Step>
  <Step title="Partners log in to access the dashboard">
    Partners authenticate with their WordPress credentials. Once logged in, the full Botky interface renders in place of the login prompt.
  </Step>
</Steps>

<Tip>
  Use a membership plugin or WordPress role restrictions to limit who can access the dashboard page. Only users who have saved a valid UChat Partner API token in their Settings tab will have a fully functional dashboard — everyone else will see empty states or API errors.
</Tip>

<Note>
  The shortcode always uses the currently logged-in WordPress user's saved API token. Each partner must open the **Settings** tab inside the dashboard and configure their own token before any workspace or client operations will work.
</Note>

## Language Support

The panel automatically renders in the correct language based on your WordPress site language setting. No additional configuration is needed to switch languages — update your site language under **Settings → General → Site Language** and the panel will follow.

| WordPress Site Language | Language rendered |
|---|---|
| English (e.g. English (US), English (UK)) | English |
| Any other language (e.g. Spanish, French) | Spanish |
