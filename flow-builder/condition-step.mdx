Con el paso **Condición** puedes ir a distintos pasos o subflujos dependiendo de si se cumplen ciertas condiciones.

## Cómo funciona

<Steps>
  <Step title="Haz clic en '+ Agregar grupo de condiciones'">
    Define al menos un grupo. El grupo **"En otro caso" (Otherwise)** siempre está presente por defecto.
  </Step>
  <Step title="Crea todos los grupos que necesites">
    La comparación va de arriba hacia abajo: en cuanto un grupo coincide, el sistema va al paso que le asignaste. Si ningún grupo coincide, se activa el grupo "En otro caso".
  </Step>
  <Step title="Elige entre 'Todas las condiciones' o 'Cualquier condición'">
    Esta opción está en la esquina superior derecha de cada grupo. **Todas las condiciones** exige que se cumplan absolutamente todas las condiciones del grupo. **Cualquier condición** exige que se cumpla al menos una.
  </Step>
</Steps>

<Card type="tip">
  Aunque sea poco probable que se use, siempre es buena práctica asignarle un paso al grupo "En otro caso" — así tu flujo nunca se queda sin salida.
</Card>

## Tipos de condición según la variable

<AccordionGroup>
  <Accordion title="Etiqueta y Secuencia" icon="tag">
    El operador puede ser "está en" o "no está en" — puedes elegir varias etiquetas o secuencias a la vez.
  </Accordion>
  <Accordion title="Texto" icon="font">
    Para "tiene algún valor" y "está vacío": al borrar una variable, esta queda vacía — por lo tanto "tiene algún valor" sería falso, y "está vacío" sería verdadero.
  </Accordion>
  <Accordion title="Número" icon="hashtag">
    Todos los operadores numéricos estándar de comparación.
  </Accordion>
  <Accordion title="Fecha y hora (DateTime)" icon="clock">
    El campo "valor" siempre debe escribirse en el formato `AAAA-MM-DD HH:mm:ss`, sin importar qué formato de visualización hayas elegido para esa variable.

    <Card type="note">
      "Antes/después" compara la fecha y hora completas, mientras que "hora antes/después" solo compara la parte de la hora.
    </Card>
  </Accordion>
  <Accordion title="JSON" icon="brackets-curly">
    Incluye operadores especiales para revisar coincidencias dentro de una lista. Los 3 últimos operadores de "tiene elemento que coincide con..." requieren que el JSON sea una lista de variables simples — no funcionan si tu JSON es una lista de otros JSON.
  </Accordion>
</AccordionGroup>

## Nuevos filtros de fecha relativa

<CardGroup cols={2}>
  <Card title="Después de hace X días" icon="calendar-plus">
    Filtra usuarios cuya última actividad ocurrió hace más de X días. Por ejemplo: revisa si la fecha de "Última vez visto" es de hace más de 30 días.
  </Card>
  <Card title="Antes de hace X días" icon="calendar-minus">
    Filtra usuarios cuya última actividad ocurrió antes de X días atrás. Por ejemplo: revisa si "Última vez visto" es de hace más de 10 días.
  </Card>
  <Card title="Después de hace X minutos" icon="hourglass-start">
    Igual que la anterior, pero en minutos — útil para actividad muy reciente.
  </Card>
  <Card title="Antes de hace X minutos" icon="hourglass-end">
    Filtra actividad ocurrida antes de X minutos atrás.
  </Card>
</CardGroup>

<Card type="note">
  Útiles para dos cosas: identificar actividad reciente (para seguimientos oportunos) y analizar comportamiento pasado (para campañas de reactivación).
</Card>

## Entrada personalizada (Custom Input)

Se usa para comparar valores dentro de una variable JSON. Busca "entrada personalizada" en el campo **si**: elige un valor dentro de un JSON, y luego compáralo igual que compararías una variable de texto normal.

## Fórmula matemática

También puedes usar fórmulas directamente en el valor de una condición (igual que en "Definir valor de variable"): `+`, `-`, `*`, `e`, `pi`, `PI`, `abs()`, `min()`, `max()`, `ceil()`, `floor()`, `log()`, `pow()`, `round()`, `sqrt()`, `sin()`, `cos()`, `tan()`.

<Card type="note">
  Si la fórmula no es válida, el resultado será `0` para números y vacío para texto.
</Card>

## Campos del sistema disponibles en Condición

Botky pone a tu disposición decenas de campos del sistema listos para usar dentro de cualquier condición.

### Correo

| Condición | Qué revisa |
|---|---|
| Email Click | Si se hizo clic en un correo específico |
| Email Opened | Si se abrió un correo específico |
| Email Delivered | Si un correo específico fue entregado |
| Email Sent | Si un correo específico fue marcado como enviado |
| Opted-In for Email | Si el usuario tiene activado el permiso de correo |

### Verdadero/Falso (Booleanos)

| Condición | Qué revisa |
|---|---|
| Subscribed to bot | Si el usuario está suscrito al bot |
| Has Interacted in last 24h | Si el usuario interactuó en las últimas 24 horas |
| Is Guest User | Si es un usuario invitado |
| Is Tester | Si es un tester |
| Is Admin | Si es un administrador |
| Is assigned to agent | Si el usuario tiene un agente asignado |
| Has SSO user connected | Si el usuario tiene SSO conectado |
| Is Business Hours | Si la interacción ocurre dentro del horario laboral configurado |
| Is assigned agent online | Si el agente asignado está conectado |
| Has agents online | Si hay algún agente humano conectado en este momento |

### Numéricos

| Condición | Qué representa |
|---|---|
| Opted-In Date (minutes ago) | Minutos desde que el usuario se suscribió al bot |
| Last seen (minutes ago) | Minutos desde que el usuario vio el último mensaje del bot |
| Last interaction (minutes ago) | Minutos desde la última interacción del usuario con el flujo |
| Last agent interaction (minutes ago) | Minutos desde la última interacción de un agente humano con el usuario |
| Opted-In for Email (minutes ago) | Minutos desde que se activó el permiso de correo |
| Opted-In for SMS (minutes ago) | Minutos desde que se activó el permiso de SMS |
| Shopping cart total price | Precio total del carrito, guardado en el campo del sistema ORDER |
| Agent Id | ID del sistema del agente humano |

### Fecha y hora

| Condición | Qué representa |
|---|---|
| Opted-In Date | Fecha en la que el usuario activó el permiso |
| Last Seen | Fecha y hora de la última vez que el usuario vio un mensaje del bot |
| BOT_CURRENT_TIME | Hora actual según la zona horaria del espacio de trabajo |
| Created At | Fecha y hora de creación del usuario |
| Last Interaction | Fecha y hora de la última interacción del usuario |
| NOW | Hora actual en UTC |
| Subscribed | Fecha y hora en la que el usuario se suscribió al bot |
| TODAY | Fecha de hoy |
