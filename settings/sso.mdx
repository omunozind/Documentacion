El **Single Sign-On (SSO)** nos da la capacidad de tener un **usuario maestro** del bot que sincroniza los datos entre todos tus usuarios del bot en distintos canales. Ayuda enormemente a sincronizar datos e identificar usuarios cuando hay varios canales involucrados.

## Habilitar el login SSO

Habilita el SSO yendo a la pestaña **Configuración** (abajo a la izquierda del menú lateral del Constructor de Flujos) y haciendo clic en la opción **SSO Settings**.

## Configurar el flujo de login SSO

Haz clic en la pestaña **Basic** y completa los datos requeridos.

Con el SSO habilitado, empezarás a ver **"SSO Required"** en los nodos de inicio (Start) de los subflujos. Al activar esta opción, se dirige al usuario al login de SSO y, tras un login exitoso, se le redirige al subflujo que seleccionaste en los ajustes.

## Sincronización de datos entre usuarios

<Frame caption="Sincronización de datos entre canales">
  <Placeholder />
</Frame>

El usuario del bot con la etiqueta **SSO** es el usuario maestro que sincroniza los datos entre el mismo usuario en distintos canales.

- Si agregas una etiqueta o cambias el valor de una variable para el usuario SSO, el cambio también se aplica a los usuarios en otros canales.
- Si agregas una etiqueta o cambias el valor de una variable para el usuario en otros canales, el cambio también se aplica al usuario SSO.

Esto permite sincronizar datos entre múltiples canales y da más flexibilidad al trabajar sobre varios canales.

<Card type="note">
  Los usuarios del bot se siguen contando por separado. El SSO solo sirve para sincronizar los datos entre usuarios; no los acumula en un solo usuario.
</Card>
