Los **Audit Logs** le dan a los usuarios la capacidad de dar seguimiento a los cambios que ocurren en su workspace. La información incluye qué miembros del equipo cambiaron de workspace, qué flujos se modificaron, qué nodos se eliminaron/crearon y quién hizo los cambios.

<Frame><Placeholder /></Frame>

Los Audit Logs ofrecen un análisis completo de la actividad que ocurre en los workspaces y brindan una capa adicional de seguridad y responsabilidad al gestionar un equipo de miembros.

<Frame><Placeholder /></Frame>

Puedes usar los filtros en la parte superior izquierda de la interfaz para filtrar por usuarios, acción y sujeto, y así monitorear cambios específicos que ocurran en un flujo o dentro del workspace.

## Lista de acciones (Actions List)

| Acción | Explicación |
|--------|-------------|
| create | Significa que se está creando un nodo, como un nodo de acción para realizar una acción o un nodo de envío de mensaje para enviar un mensaje |
| delete | Significa que algo se elimina. Puede ser un nodo, un disparador, un CUF o un usuario del bot |
| install | Significa que se instaló una plantilla o un mini-app en el bot |
| link | Significa que se vinculó una integración con el workspace. Un ejemplo sería una conexión SMTP o una integración de OpenAI |
| login | Significa que un usuario inició sesión en el workspace |
| logout | Significa que un usuario cerró sesión en el workspace |
| switch | Significa que un usuario cambió de un workspace a otro |
| system | Significa que hubo un evento del sistema |
| uninstall | Significa que se desinstaló un mini-app del workspace |
| unlink | Significa que se desvinculó una integración del workspace |
| update | Significa que se actualizó un nodo; un ejemplo sería actualizar el texto de un nodo Send Message |
